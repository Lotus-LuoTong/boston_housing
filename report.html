问题 1 - 特征观察¶

如前文所述，本项目中我们关注的是其中三个值:'RM'、'LSTAT' 和'PTRATIO'，对每一个数据点:

  • 'RM' 是该地区中每个房屋的平均房间数量；
  • 'LSTAT' 是指该地区有多少百分比的房东属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

提示：你预期一个'RM' 值是6的房屋跟'RM' 值是7的房屋相比，价值更高还是更低呢？

问题 1 - 回答：¶

  • 'RM'的增大会导致'MEDV'增大，因为根据经验，每个房屋的房间数量越多，其价格也会越贵，因此每个房屋的平均房间数量的增大会引起房屋价格的增加
  • 'LSTAT'的增大会导致'MDEV'减小，由经验可知，一个地区的高收入人群越大，房价越高，而'LSAT'的增大反映的是低收入人群的增多和高收入人群的减少，因此会引起房屋价格降低
  • 'PTRATIO'的增大会导致'MEDV'的降低，但不是显著地降低，因为学生与老师的数目比一定程度上反映了该地区居民的受教育程度和收入水平（受教育程度和收入水平越高，生育意愿越低，学生也就越少），因此增大'PTRATIO'表明学生相对数量增多，受教育水平和收入水平降低，因而房价也会降低；另一方面，'PTRATIO'并不是直接影响房价的因素，而是通过影响其他相关的变量间接地影响房价，因此不会对'MEDV'造成太大的影响。
  • 实际上，上面所说的某个变量“导致”房价变化是不准确的，因为严格地说，它们之间是否存在因果关系需要经过检验，而在此之前，我们只能说两个变量之间具有相关性。

编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    from sklearn.metrics import r2_score
    score = r2_score(y_true, y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    true_mean = np.mean(y_true)
    tot = [(x - true_mean) ** 2 for x in y_true]
    res = [(x - y) ** 2 for (x,y) in zip(y_predict, y_true)] 
    ss_tot = sum(tot)
    ss_res = sum(res)
    score = 1 - float(ss_res) / ss_tot

    return score

什么是网格搜索法？如何用它来优化模型？

问题 7 - 回答:¶

  • 网格搜索法是对机器学习模型进行调参数的时候，给定每个参数可能的取值和模型评价函数，对这些取值的组合进行遍历，每次取一组参数训练模型，用给定的评价函数计算打分，最后取打分最高的一组参数作为最佳参数。
  • 网格搜索法需要首先确定需要优化的参数，列出每个参数可能的取值，列出所有可能参数的组合，分别用这些参数组合训练模型，并使用交叉验证对模型的效果进行评估，选出最优的参数组合。

  1. K折交叉验证法是评估模型泛化能力的一种方法。将训练数据按顺序平均划分为K份（默认情况下不打乱顺序，也可指定参数使其随机划分）然后进行遍历，每次取其中一份作为测试集，其余的K-1份作为训练集训练模型，计算模型在测试集的打分。K次训练中测试集打分的均值可以用来评价模型的泛化能力。
  2. GridSearchCV中可以指定K折交叉验证的折数，首先将数据集随机平均划分为K份并进行遍历，每次取其中一份作为测试集，其余K-1份作为训练集，分别使用每个参数组合训练模型，记录每个参数组合在该折中的表现，最后对取每个参数组合的打分在K次训练中的均值作为最终的打分，打分最高的一组参数就是最佳参数。
  3. 'cv_results_'包含了以下信息：
    1. 所有可能的参数组合以及参数在该组合中的状态（mask）
    2. K折交叉验证中每一折每个模型在测试集上的打分
    3. 每个模型测试集打分的均值、标准差
    4. K折交叉验证中每一折每个模型在训练集上的打分
    5. 每个模型训练集打分的均值、标准差
    6. 每个模型在K折交叉验证中训练时间的均值和标准差
    7. 每个模型在K折交叉验证中打分时间的均值和标准差
    8. 输入参数
    1. 如果不使用交叉验证，而仅仅是将数据集划分为测试集和训练集，划分的结果具有偶然性。将这种方法用于网格搜索，可能陷入局部最优解，即搜索到的最优参数刚好在本次划分的训练集和测试集上具有较好的表现。
    2. 使用K折交叉验证，将每次训练得到的打分取平均值，有效地避免了单次划分数据集的偶然性，更容易得到全局最优参数。

def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    from sklearn.model_selection import KFold
    from sklearn.tree import DecisionTreeRegressor
    from sklearn.metrics import make_scorer
    from sklearn.model_selection import GridSearchCV
    cross_validator = KFold(random_state = 6)
    
    regressor = DecisionTreeRegressor()

    params = {'max_depth':range(1,11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, params, scoring_fnc, cv = cross_validator)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    # 返回网格搜索后的最优模型
    return grid.best_estimator_

模型健壮性¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
